# 02. Methodology

## 1. Define the travel context

Before collecting sources, define the mission context:

| Field | Example |
|---|---|
| Destination | Port Azure, Republic of Lydora |
| Travel purpose | Institutional meeting / short business visit |
| Traveller profile | Adult staff member, no special risk profile assumed |
| Duration | 3 nights |
| Movement pattern | Airport → hotel → meeting venue → hotel → airport |
| Main need | Pre-travel briefing and accommodation mapping |

This prevents open-ended research and keeps the assessment proportionate.

## 2. Identify risk categories

The project uses six baseline categories:

1. **Crime and personal security**
2. **Civil unrest and political tension**
3. **Transport and airport transfers**
4. **Health and medical access**
5. **Local law and administrative constraints**
6. **Information / communication risks**

For each category, the analyst records:

- what is known;
- source strength;
- uncertainty level;
- relevance to the traveller;
- mitigation or escalation note.

## 3. Collect public sources

Sources are logged in `data/sample_source_log.csv` using the following fields:

- source name;
- source type;
- date checked;
- topic;
- key finding;
- reliability rating;
- limitation;
- action or follow-up.

The workflow favours official sources, institutional sources, and corroborated public reporting.

## 4. Rate source reliability

The project uses a simple A-D source scale:

| Rating | Meaning |
|---|---|
| A | Official or primary source; high reliability for the specific claim. |
| B | Reputable secondary source or established specialist source. |
| C | Useful contextual source, but requires corroboration. |
| D | Low reliability, unclear provenance, or unsuitable for decision-making. |

A reliability rating does not mean the source is always right. It only describes how much weight it should receive for the specific assessment.

## 5. Map accommodations

Accommodation options are scored using practical factors:

- distance to meeting venue;
- distance to airport;
- access control;
- reception coverage;
- neighbourhood risk note;
- transport availability;
- cancellation flexibility;
- emergency access;
- source confidence.

The aim is not to rank luxury or comfort. The aim is to support safer, more auditable travel planning.

## 6. Monitor public signals ethically

The monitoring framework is limited to:

- official public alerts;
- verified public institutional posts;
- public transport disruption channels;
- public event notices;
- mainstream news and confirmed crisis updates.

It avoids collecting personal data, profiling private individuals, or treating unverified posts as facts.

## 7. Analyse traveller feedback

Feedback is grouped into themes:

- transport delays;
- hotel access concerns;
- harassment / unsafe feeling;
- document checks;
- medical access concerns;
- communication issues.

Each theme is converted into:

- evidence summary;
- severity;
- recurrence;
- operational recommendation;
- whether escalation is needed.

## 8. Triage urgent communications

Urgent messages are separated into:

- routine;
- watchlist;
- urgent review;
- immediate escalation.

The triage is based on content, specificity, imminence, credibility, and relevance to current travel.

## 9. Produce the brief

The final briefing should be:

- concise;
- source-aware;
- dated;
- specific about uncertainty;
- practical;
- non-alarmist;
- clear about escalation triggers.
